### Phase 3: The Deep Work Mission (Module 1)

**Mission Title:** The FinTech Server Boot Sequence

**The Scenario**
You are the junior software engineer on duty at a major payment processing company. The main transaction server just crashed. The automated reboot script was written by an intern and it is completely broken.

The security firewall requires a very specific sequence of system logs to authorize the reboot. If the execution order is wrong, or if a warning is logged as a standard message, the firewall will permanently lock the system. You have 10 minutes to fix the script before millions of transactions fail.

**The Broken Codebase (What the user sees)**

```javascript
console.log("Server reboot initiated.");
/* Check power supply */
console.error("Connecting to main database...");
console.log(System diagnostic complete);

// The next line is a broken statement
console.log("Database connection failed. Retrying...");

console.warn("Power supply unstable.");

```

**The Mission Objectives (Strict Requirements)**
You must edit the code to satisfy the following security rules exactly in this order:

1. **Execution Order:** The system must check the power supply before it connects to the database.
2. **Console Types:** * Standard steps must be normal logs.
* The unstable power supply must be a warning.
* The database connection failure must be a critical error.


3. **Statements vs Expressions:** The line logging the diagnostic status is currently broken because it lacks quotes (treating text as undefined variables instead of a string expression). Fix this expression.
4. **Comments:** You must write one single-line comment directly above the warning message explaining *why* the warning is happening. You must also write one multi-line comment at the very bottom of the file stating the server status is pending.

**The Hidden Test Cases (System Validation)**

* **Test Case 1:** Intercept the console output. Verify that the exact first string printed is `"Server reboot initiated."` via `console.log`.
* **Test Case 2:** Verify that the second output is exactly `"Power supply unstable."` and it is strictly called using `console.warn`.
* **Test Case 3:** Verify that the third output is exactly `"Connecting to main database..."` via `console.log`.
* **Test Case 4:** Verify that the fourth output is exactly `"Database connection failed. Retrying..."` and it is strictly called using `console.error`.
* **Test Case 5:** Verify the broken expression is fixed and prints `"System diagnostic complete"` via `console.log` as the final output.
* **Test Case 6:** Parse the AST (Abstract Syntax Tree) to confirm the existence of at least one single-line comment (`//`) and one multi-line comment (`/* */`) in the correct locations.

**The Reward System (Gamification Trigger)**

* **On Mission Success:** A 3D animation of a neon digital mystery box drops onto the screen.
* **Reward Output:** "Mission Accomplished! Server rebooted successfully. You have earned a Tier 1 Mystery Box."
* **Loot Reveal:** The box opens to award the user **15 Platform Coins**. The dashboard progress bar lights up, indicating they are one step closer to redeeming physical merchandise.

### Phase 3: The Deep Work Mission (Module 2)

**Mission Title:** The Ride-Share Ghost Driver

**The Scenario**
You are a backend engineer at a major ride-sharing startup. The dispatch system is currently experiencing a critical failure. Customers are requesting rides, but their apps are crashing, showing "Syntax Error," or displaying the driver's name as a blank glitch (`undefined`).

The code was hastily written during a server outage. It is filled with illegal variable names, broken memory rules, and reserved keywords that are destroying the compiler. You have 10 minutes to rewrite the memory allocation block before the company loses thousands of daily active users to a competitor.

**The Broken Codebase (What the user sees)**

```javascript
/* --- SYSTEM ALERT: DISPATCH ALLOCATION FAILING --- */

// 1. Ride Configuration
let 1stRiderPickup = "Central Station"; 
const maxPassengers = 4;
let drop_location = "International Airport";
let default = "Standard Ride";

// 2. State Management
const rideStatus = "Searching for Driver";

// System attempts to update the status when a driver is matched
rideStatus = "Driver Arriving"; // CRASH: TypeError 

// 3. Driver Allocation
let assignedDriver; 

// The app is sending the driver data to the customer's screen
console.log("Your driver is:", assignedDriver); 

```

**The Mission Objectives (Strict Requirements)**
You must refactor (clean up) the codebase to satisfy the memory engine. Fix the following issues:

1. **Illegal Variables:** `1stRiderPickup` starts with a number. Rename it to a valid variable name that represents the pickup location.
2. **Reserved Keywords:** `default` is a core JavaScript keyword and cannot be used as a variable name. Change it to represent the ride tier (e.g., standard, premium).
3. **Naming Conventions:** `drop_location` uses snake_case. Convert it to strict `camelCase`.
4. **Mutability Rules:** The system is crashing because it is trying to reassign `rideStatus`, which is locked as a `const`. Fix the memory declaration so the status can be successfully updated to `"Driver Arriving"`.
5. **The `undefined` Glitch:** The `assignedDriver` variable is declared but never assigned a value, sending `undefined` to the user's phone. Assign a driver's name (a string) to this variable *before* the `console.log` executes.

**The Hidden Test Cases (System Validation)**

* **Test Case 1 (Illegal Name Check):** AST (Abstract Syntax Tree) Parser verifies that no variable names start with a number.
* **Test Case 2 (Reserved Keyword Check):** AST Parser verifies that the reserved keyword `default` has been completely removed from the variable declarations.
* **Test Case 3 (camelCase Validation):** Regex validator scans all variable names in the script. Fails immediately if any underscore (`_`) is detected (e.g., `drop_location`).
* **Test Case 4 (Const vs Let):** AST validation checks the declaration of `rideStatus`. It must be declared using `let` so the reassignment on the following line executes without throwing a `TypeError`.
* **Test Case 5 (Undefined Prevention):** Intercept the `console.log`. Verify that the second argument printed is a valid `String` (length > 0) and strictly NOT `undefined`.
* **Test Case 6 (Strict Execution):** The code must execute from top to bottom without throwing a single compilation or syntax error.

**The Reward System (Gamification Trigger)**

* **On Mission Success:** The screen darkens, and a glowing neon green 3D Mystery Box drops into the center of the code editor.
* **Reward Output:** "System Restored! Dispatch algorithm is online. You have earned a Tier 2 Mystery Box."
* **Loot Reveal:** The box unlocks to grant **20 Platform Coins** and a permanent **"Memory Master" Badge** on their Skill Radar dashboard.
